# RepReady Google Sheet — schema contract

This is the **shared contract** between three things:

1. The **Google Sheet** (the athlete data store, read/written by Claude's Drive tool).
2. The **RepReady MCP server** (`resources/repready_mcp/server.py`) — its tools parse data in this shape.
3. The **`/repready` skill** (`resources/repready_skill/SKILL.md`) — it reads/writes rows in this shape.

The workbook has **5 tabs**: `Athletes`, `WorkoutHistory`, `Benchmarks`, `TrainingRules`, `DemoPrompts`.

> **JSON-string rule** (from the sample-data README): every list/nested field is stored as a
> **JSON string** inside a single cell, so tools parse it deterministically with `json.loads`.
> Example cell value: `["ski_erg","sled","rower"]` or `[{"area":"left_shin","status":"managing"}]`.

> **Privacy rule:** every tool takes a **trusted `active_user_id`** (passed by the caller, never
> derived from chat text). Tools only ever return / mutate rows whose `user_id == active_user_id`.

Seed CSVs (one per tab) are generated by `generate_seed.py` into this folder. Import each CSV as
its own tab. To swap in Hazem's real data later, replace the dicts in `generate_seed.py` (or just
re-import real CSVs with the same columns) — the column names are the contract.

## Tab: Athletes

One row per athlete. `user_id` is the trusted scope key.

| Column | Type | Notes |
|---|---|---|
| `user_id` | string | Trusted scope key, e.g. `ath_amelia`. Tools filter on this. |
| `name` | string | Display name. |
| `division` | string | `Open` \| `Pro` \| `Doubles`. |
| `gender` | string | `M` \| `F`. |
| `age` | number | |
| `age_group` | string | HYROX age band, e.g. `30-34`. |
| `body_weight_kg` | number | |
| `height_cm` | number | |
| `goal_event` | string | e.g. `HYROX London`. |
| `goal_event_date` | date | ISO `YYYY-MM-DD`. |
| `goal_finish_time` | string | `HH:MM:SS`. |
| `current_pb` | string | `HH:MM:SS` or empty. |
| `training_days_per_week` | number | |
| `equipment_access` | **JSON string** | list, e.g. `["ski_erg","sled","rower","wall_ball","kettlebell","running_track"]`. |
| `injuries` | **JSON string** | list of `{area,status,notes}`; `status` ∈ `managing`/`recovering`/`cleared`. |
| `experience_level` | string | `beginner` \| `intermediate` \| `advanced`. |
| `notes` | string | Free text. |

## Tab: WorkoutHistory

One row per session. Seed spans **2026-06-01 → 2026-06-13**.

| Column | Type | Notes |
|---|---|---|
| `log_id` | string | Unique, e.g. `wh_amelia_0601`. |
| `user_id` | string | FK to Athletes. |
| `date` | date | ISO `YYYY-MM-DD`. |
| `session_type` | string | `run` \| `intervals` \| `strength` \| `station_work` \| `hyrox_sim` \| `recovery` \| `rest`. |
| `title` | string | Short label. |
| `details` | **JSON string** | session-specific, e.g. `{"run":{"distance_km":8,"time_sec":2400},"stations":[{"station":"sled_push_50m","weight_kg":102,"time_sec":78}]}`. |
| `duration_min` | number | |
| `distance_km` | number | `0` if N/A. |
| `avg_rpe` | number | 1–10 perceived exertion. |
| `pain_flags` | **JSON string** | list of body areas, e.g. `["left_shin"]` or `[]`. |
| `completed` | boolean | `TRUE`/`FALSE`. |
| `notes` | string | |

## Tab: Benchmarks

HYROX station standards (reference data, **also embedded in `server.py`** so `get_benchmarks`
works without Sheet access). Keep this tab in sync with the server's `BENCHMARKS` table.

| Column | Type | Notes |
|---|---|---|
| `benchmark_id` | string | e.g. `bm_open_m_sled_push`. |
| `division` | string | `Open` \| `Pro`. |
| `gender` | string | `M` \| `F`. |
| `station` | string | canonical station id (see station ids below). |
| `weight_kg` | number | load for weighted stations; `0` for runs/erg/row. |
| `reps` | number | rep target (wall balls); `0` otherwise. |
| `elite_time_sec` | number | top-tier split. |
| `competitive_time_sec` | number | strong age-grouper split. |
| `intermediate_time_sec` | number | mid-pack split. |
| `beginner_time_sec` | number | first-timer split. |
| `notes` | string | |

**Canonical station ids:** `1km_run`, `ski_erg_1000m`, `sled_push_50m`, `sled_pull_50m`,
`burpee_broad_jump_80m`, `row_1000m`, `farmers_carry_200m`, `sandbag_lunge_100m`, `wall_balls`.

## Tab: TrainingRules

Deterministic rules powering `validate_training_plan` and `check_request_safety` (the server
embeds equivalent logic; this tab documents it and lets non-engineers tune thresholds).

| Column | Type | Notes |
|---|---|---|
| `rule_id` | string | e.g. `rule_max_hi_sessions`. |
| `category` | string | `safety` \| `volume` \| `intensity` \| `recovery` \| `privacy` \| `scope`. |
| `name` | string | Human label. |
| `severity` | string | `block` \| `warn` \| `info`. |
| `description` | string | What it checks and why. |
| `params` | **JSON string** | thresholds, e.g. `{"max_hi_sessions":3}`. |
| `message` | string | What the coach tells the user when it triggers. |
